# Sysmon (System Monitor) 🚀

Легковесная клиент-серверная система для мониторинга ресурсов Linux-серверов. 
Проект состоит из Агента (собирает метрики) и Центрального сервера (принимает данные и сохраняет их в базу).

## 🏗 Архитектура

* **Agent**: Работает в фоне на целевых машинах, собирает данные (ОС, CPU, RAM) и отправляет их по HTTP.
* **Server**: REST API на Go с чистой слоистой архитектурой (`handlers`, `models`, `storage`).
* **Database**: PostgreSQL для надежного хранения истории метрик.

## ✨ Ключевые особенности

* **Clean Architecture**: Жесткое разделение логики обработки HTTP-запросов и работы с базой данных.
* **Graceful Shutdown**: Безопасное завершение работы сервера без потери данных при получении системных сигналов (SIGINT/SIGTERM).
* **Dependency Injection**: Прокидывание пула соединений БД в слой хендлеров.
* **Dockerized**: Сервер и БД полностью упакованы в контейнеры (используется Multi-stage build для минимизации размера образа Go).

## 🚀 Быстрый старт (Сервер)

Для запуска центрального узла потребуется только Docker и Docker Compose.

1. Склонируйте репозиторий:
```bash
git clone [https://github.com/mintrage/sysmon.git](https://github.com/ВАШ_ЮЗЕРНЕЙМ/sysmon.git)
cd sysmon
```

2. Запустите инфраструктуру одной командой:
```bash
docker compose up --build -d
```

3. Проверьте логи сервера:
```bash
docker compose logs -f server
```

Сервер поднимется и будет готов принимать входящие метрики на порту `8080`.

## 📡 API Эндпоинты

* `POST /api/metrics` — Прием метрик от агента (ожидает JSON формат).
* `GET /api/metrics/latest` — Получение самой свежей записи из базы данных.
